Module 2 · Speed Calculation

Object Speed Estimation

Tracked detections give us pixel positions over time. We need to convert that to real-world speed.

Step 1 — Pixel Velocity

Δpx = center(t) − center(t−n)
dt = n × frame_interval
v_px = Δpx / dt px/s

Step 2 — Real-world

v_rel = v_px × scale
v_abs = v_rel + v_own
(from CAN-bus)

Step 3 — Units

m/s → knots × 1.944
m/s → kph × 3.6
configurable
✓
📏

Fixed Scale Ratio

User defines meters_per_pixel at a reference distance. Simple, fast, good enough for open-sea single-range scenarios.

Recommended (MVP)
✓
📐

Perspective Transform

4-point homography mapping pixel coords to real-world coords. Much more accurate when camera angle is known. Requires calibration step.

Needs calibration
✓
📦

Bounding Box Height Proxy

Estimate distance from bbox height (larger = closer). Works without calibration — uses known average object size (e.g. 10m boat).

No calibration
🔢 Buffer depth: We maintain a deque of the last N=10 positions per tracker_id and compute velocity over the full window for stability. Configurable.

Simulated (Random Walk)

Values drift realistically using random walk. Heading drifts ±2°/s, speed ±0.1kn/s, RPM proportional to speed. Feels alive.
